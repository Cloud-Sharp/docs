# 기능 요구사항 상세

|SFR ID|기능명|설명|입력|출력|예외|우선순위|
|---|---|---|---|---|---|---|
|SFR-001|회원가입|비회원은 이메일 또는 아이디와 비밀번호를 이용해 계정을 생성할 수 있어야 한다. 시스템은 계정 정보를 저장하고 이후 로그인 가능 상태로 전환해야 한다.|이메일 또는 아이디, 비밀번호|회원가입 성공/실패 결과, 생성된 사용자 정보|이미 존재하는 계정, 비밀번호 정책 미충족, 필수값 누락|높음|
|SFR-002|로그인|사용자는 등록된 계정 정보로 로그인할 수 있어야 하며, 로그인 성공 시 인증 상태를 획득해야 한다.|이메일 또는 아이디, 비밀번호|로그인 성공/실패 결과, JWT Bearer 토큰|계정 없음, 비밀번호 불일치, 비활성 계정|높음|
|SFR-003|로그아웃|로그인한 사용자는 현재 인증 토큰을 종료할 수 있어야 한다.|`Authorization` 헤더의 JWT Bearer 토큰|로그아웃 성공/실패 결과|인증 정보 없음, 이미 만료된 토큰|높음|
|SFR-004|인증 세션 유지|로그인한 사용자는 재인증 없이 보호된 API를 사용할 수 있어야 하며, 시스템은 인증 상태를 유지해야 한다.|`Authorization` 헤더의 JWT Bearer 토큰|인증된 사용자 컨텍스트, 보호 API 접근 허용|만료된 토큰, 위조 또는 무효 토큰|높음|
|SFR-005|기본 프로필 조회|로그인한 사용자는 자신의 기본 프로필 정보를 조회할 수 있어야 한다.|인증 정보|사용자 프로필 정보|인증 실패, 존재하지 않는 사용자|중간|
|SFR-006|Space 목록 조회|로그인한 사용자는 자신이 생성했거나 초대를 통해 참여 중인 Space 목록을 조회할 수 있어야 한다.|인증 정보|Space 목록, 내 Role, 상태 정보|인증 실패|높음|
|SFR-007|Space 생성|로그인한 사용자는 새로운 Space를 생성할 수 있어야 하며, 생성 즉시 Owner 권한을 가져야 한다.|Space 이름, 설명(선택), 초기 용량 정책(선택)|생성된 Space 정보, 생성자 Role 정보|인증 실패, 이름 정책 위반, 생성 실패|높음|
|SFR-008|Space 상세 조회|사용자는 자신이 접근 가능한 특정 Space의 상세 정보와 기본 정책을 조회할 수 있어야 한다.|Space ID, 인증 정보|Space 상세 정보, Quota, 내 Role|대상 Space 없음, 접근 권한 없음|높음|
|SFR-009|Space 이름 변경|Owner 또는 권한 있는 사용자는 Space 이름을 변경할 수 있어야 한다.|Space ID, 새 이름|변경된 Space 정보|대상 Space 없음, 권한 없음, 이름 정책 위반|중간|
|SFR-010|Space 삭제|Owner는 Space를 삭제하거나 비활성화할 수 있어야 한다. 삭제 정책은 시스템 정책에 따라 소프트 삭제 또는 아카이브로 처리할 수 있어야 한다.|Space ID|삭제/비활성화 결과|대상 Space 없음, 권한 없음, 삭제 불가 상태|중간|
|SFR-011|Space 초대 생성|Owner 또는 권한 있는 사용자는 다른 사용자를 Space에 초대할 수 있어야 한다.|Space ID, 초대 대상 사용자 또는 이메일, 부여 Role, 만료일(선택)|초대 생성 결과, 초대 토큰 또는 초대 정보|대상 Space 없음, 권한 없음, 잘못된 대상, 중복 초대|높음|
|SFR-012|Space 초대 수락|초대받은 사용자는 유효한 초대를 수락하여 Space 멤버가 될 수 있어야 한다.|초대 토큰 또는 초대 ID, 인증 정보|Space 참여 결과, 멤버십 정보|만료된 초대, 이미 수락된 초대, 인증 실패|높음|
|SFR-013|Space 멤버 목록 조회|권한 있는 사용자는 특정 Space의 멤버 목록과 각 멤버의 Role을 조회할 수 있어야 한다.|Space ID, 인증 정보|멤버 목록, Role, 상태|대상 Space 없음, 권한 없음|높음|
|SFR-014|Space 멤버 권한 변경|Owner 또는 권한 있는 사용자는 Space 멤버의 Role을 변경할 수 있어야 한다.|Space ID, 대상 사용자 ID, 새 Role|변경된 멤버 정보|대상 멤버 없음, 권한 없음, 허용되지 않는 Role 변경|높음|
|SFR-015|Space 멤버 제거|Owner 또는 권한 있는 사용자는 Space 멤버를 제거할 수 있어야 한다.|Space ID, 대상 사용자 ID|제거 결과|대상 멤버 없음, 권한 없음, Owner 제거 시도|중간|
|SFR-016|Space Quota 조회|권한 있는 사용자는 특정 Space의 총 허용 용량, 사용량, 예약 용량을 조회할 수 있어야 한다.|Space ID, 인증 정보|총 허용 용량, 사용량, 예약 용량, 사용률|대상 Space 없음, 권한 없음, 집계 실패|높음|
|SFR-017|Space Quota 설정/변경|Owner 또는 권한 있는 사용자는 Space의 Quota를 설정하거나 변경할 수 있어야 한다.|Space ID, 변경할 용량 값|변경된 Quota 정보|대상 Space 없음, 권한 없음, 현재 사용량보다 작은 값 설정 시도|중간|
|SFR-018|폴더 목록 조회|사용자는 특정 Space 안의 특정 폴더에 포함된 파일 및 하위 폴더 목록을 조회할 수 있어야 한다.|Space ID, 폴더 ID, 인증 정보|파일/폴더 목록|대상 Space 없음, 대상 폴더 없음, 권한 없음|높음|
|SFR-019|폴더 생성|사용자는 특정 Space의 특정 부모 폴더 하위에 새 폴더를 생성할 수 있어야 한다.|Space ID, 부모 폴더 ID, 폴더명|생성된 폴더 정보|대상 Space 없음, 부모 폴더 없음, 권한 없음, 중복 이름 정책 위반|높음|
|SFR-020|폴더명 변경|사용자는 Space 내 기존 폴더의 이름을 변경할 수 있어야 한다. 실제 저장 경로가 아니라 메타데이터 기준으로 처리한다.|Space ID, 폴더 ID, 새 폴더명|변경된 폴더 정보|대상 폴더 없음, 권한 없음, 중복 이름 정책 위반|중간|
|SFR-021|폴더 이동|사용자는 Space 내 폴더를 다른 상위 폴더로 이동할 수 있어야 한다. 실제 저장 객체 이동이 아니라 메타데이터 기준으로 처리한다.|Space ID, 폴더 ID, 대상 부모 폴더 ID|이동된 폴더 정보|대상 폴더 없음, 대상 부모 폴더 없음, 권한 없음, 순환 이동 시도|중간|
|SFR-022|폴더 삭제|사용자는 Space 내 폴더를 삭제할 수 있어야 한다. 하위 항목 존재 시 시스템 정책에 따라 일괄 삭제 또는 삭제 제한으로 처리한다.|Space ID, 폴더 ID|삭제 성공/실패 결과|대상 폴더 없음, 권한 없음, 루트 폴더 삭제 시도|중간|
|SFR-023|파일 업로드 초기화|사용자는 특정 Space와 대상 폴더를 기준으로 파일 업로드를 시작할 수 있어야 한다. 시스템은 업로드 세션과 업로드 URL을 발급해야 한다.|Space ID, 대상 폴더 ID, 파일명, 크기, 타입|업로드 세션 정보, 업로드 URL|권한 없음, quota 초과, 파일 크기 제한 초과, 대상 폴더 없음|높음|
|SFR-024|파일 업로드 진행 및 재개|시스템은 대용량 파일 업로드 중 네트워크가 끊겨도 이어서 업로드할 수 있어야 한다.|tus 업로드 세션, 청크 데이터|업로드 진행 상태, 재개 가능 상태|세션 만료, 손상된 청크, 잘못된 업로드 오프셋|높음|
|SFR-025|업로드 완료 처리|파일 업로드가 완료되면 시스템은 파일 무결성을 검증하고 메타데이터를 저장한 뒤 Space 내 파일 목록에 반영해야 한다.|업로드 완료 이벤트, 파일 식별 정보|저장된 파일 메타데이터, 목록 반영 결과|저장 실패, finalize 실패, 파일 무결성 오류|높음|
|SFR-026|파일 다운로드|권한 있는 Space 멤버는 저장된 파일에 대한 다운로드 세션을 발급받고 실제 파일을 내려받을 수 있어야 한다.|Space ID, 파일 ID, 인증 정보|다운로드 세션 또는 파일 다운로드 응답|권한 없음, 대상 파일 없음, 삭제/격리된 파일|높음|
|SFR-027|파일명 변경|사용자는 Space 내 파일명을 변경할 수 있어야 한다. 실제 저장 객체는 유지하고 메타데이터만 수정한다.|Space ID, 파일 ID, 새 파일명|변경된 파일 정보|대상 파일 없음, 권한 없음, 중복 이름 정책 위반|중간|

|SFR-029|파일 삭제|사용자는 Space 내 파일을 삭제할 수 있어야 하며, 삭제된 파일은 목록에서 제외되어야 한다.|Space ID, 파일 ID|삭제 성공/실패 결과|대상 파일 없음, 권한 없음, 이미 삭제된 파일|높음|
|SFR-030|목록 정렬|사용자는 특정 Space/폴더의 파일 및 폴더 목록을 이름, 크기, 수정일 기준으로 정렬할 수 있어야 한다.|Space ID, 폴더 ID, 정렬 기준, 정렬 방향|정렬된 파일/폴더 목록|지원하지 않는 정렬 기준, 대상 폴더 없음|중간|
