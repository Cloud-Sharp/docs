

### 권한 박탈 및 공유 링크 폐기 시 즉시 Revoke 지원 여부

본 시스템은 **다운로드 세션 발급 이후 권한 박탈 또는 공유 링크 폐기 시, 이미 발급된 세션에 대한 즉시 revoke를 지원하지 않는다.**

즉, 다음 상황이 발생하더라도 이미 발급된 다운로드 세션은 남은 TTL 동안 유효하다.

- 인증 사용자의 파일 접근 권한이 세션 발급 직후 박탈된 경우
    
- 공유 링크가 세션 발급 직후 비활성화되거나 폐기된 경우
    
- 다운로드 허용 정책이 세션 발급 이후 변경된 경우
    

이 정책은 기존 문서의 “다운로드 세션은 발급 시점 권한을 기준으로 TTL 동안 유효”라는 원칙을 그대로 유지하는 결정이다.

### 즉시 Revoke 미지원 근거

즉시 revoke를 기본 동작으로 두지 않는 이유는 다운로드 세션이 일반적인 API access token이 아니라, **브라우저 다운로드 동작을 수용하기 위한 짧은 수명의 스트리밍 세션**이기 때문이다. 기존 문서에서도 세션은 짧게 유지하되, 세션이 살아 있는 동안은 브라우저의 동일 URL 재요청, `Range` 기반 이어받기, 네트워크 재시도를 허용하도록 설계되어 있다. 다만 한 HTTP 요청 안의 multi-range와 병렬 Range 다운로드는 지원하지 않는다. 이 전제 아래에서는 “발급 직후 즉시 무효화 가능” 정책이 브라우저 호환성과 충돌할 수 있다.

또한 기존 설계는 세션을 **파일 ID + 요청자 식별자에 바인딩**하고, **짧은 TTL**과 **재요청 허용**을 조합하여 재생 공격 위험을 줄이는 구조를 택하고 있다. 즉, 본 정책은 “세션을 길게 유지하면서 중앙에서 강제 차단하는 방식”이 아니라, “세션 자체를 짧게 유지해 노출 시간을 제한하는 방식”을 따른다.

### 최대 허용 노출 시간

즉시 revoke를 지원하지 않는 대신, 다운로드 세션의 최대 허용 노출 시간은 **5분**으로 제한한다.

즉, 세션 발급 이후 권한이 박탈되거나 공유 링크가 폐기되더라도, 이미 발급된 세션은 최대 5분 이내에서만 유효하다. 5분이 경과하면 세션은 자동 만료되며, 이후 동일 요청자도 다시 권한 검증, 다운로드 가능 상태 사전 검증, 세션 발급 단계를 거쳐야 한다. 기존 문서의 다운로드 흐름이 “권한 검증 → 사전 검증 → 세션 발급 → 스트리밍” 구조이므로, 세션 만료 이후에는 재발급 전 권한이 다시 평가된다.

### TTL 정책 확정

- 세션은 단명 토큰으로 운영한다.
    
- 세션은 파일 ID와 요청자 식별자에 바인딩한다.
    
- 세션 TTL 내 반복 GET / Range / resume 요청은 허용한다.
    
- 세션 만료 후에는 반드시 재검증을 거쳐 새 세션을 발급받아야 한다.
    

즉, 본 정책은 “즉시 revoke 미지원”을 선택하되, 그 대신 TTL을 명시적으로 제한하여 세션 노출 시간을 통제하는 방식이다.

### 보안 사고 대응 절차 내 강제 만료 단계 포함 여부

현재 보안 사고 대응 절차에는 **기존 다운로드 세션을 강제로 만료시키는 별도 단계는 포함하지 않는다.**

즉, 현재 운영 정책에서는 다음과 같은 중앙 강제 조치를 기본 기능으로 제공하지 않는다.

- 특정 사용자 세션 일괄 무효화
    
- 특정 공유 링크에 의해 발급된 다운로드 세션 일괄 차단
    
- 특정 파일에 대한 활성 다운로드 세션 전체 강제 만료
    

따라서 보안 사고 발생 시에도 이미 발급된 다운로드 세션은 TTL 종료 전까지는 유효할 수 있으며, 운영자는 링크 폐기나 권한 회수만 수행하고 세션 자체는 자연 만료에 의존하게 된다. 이 점은 현재 정책의 명확한 한계로 간주해야 한다. 기존 문서에도 세션 메타데이터를 Redis/DB에 선택적으로 둘 수 있다고만 되어 있을 뿐, 강제 revoke 절차는 정의되어 있지 않다.

### 운영 원칙 요약

본 시스템의 다운로드 세션 권한 변경 및 revoke 정책은 다음 원칙을 따른다.

- 다운로드 세션은 발급 시점 권한을 기준으로 TTL 동안 유효하다.
    
- 세션 발급 후 권한 박탈이나 공유 링크 폐기가 발생해도 즉시 revoke는 지원하지 않는다.
    
- 즉시 revoke 미지원에 대한 보완책으로 세션 TTL은 최대 5분으로 제한한다.
    
- 세션 만료 후 재다운로드 시에는 권한 및 파일 상태를 다시 검증한다.
    
- 현재 보안 사고 대응 절차에는 활성 다운로드 세션 강제 만료 단계가 포함되지 않는다.
