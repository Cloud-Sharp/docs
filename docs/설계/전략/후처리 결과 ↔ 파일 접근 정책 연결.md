## 후처리 결과와 파일 접근 정책 연결

본 시스템은 `FileItem` 생성 완료 후 이미지 썸네일 생성, 텍스트/PDF 미리보기 캐시, 바이러스 스캔, 검색 인덱싱, 감사 로그 기록을 후처리 단계에서 수행한다. 기존 업로드 파이프라인은 이들 작업을 큐 기반 또는 비동기 방식으로 처리하도록 정의하고 있으며, 클라이언트에도 업로드 결과와 함께 후처리 상태를 제공하도록 되어 있다.

동시에 다운로드 파이프라인은 세션 발급 전 사전 검증 단계에서 파일 메타데이터, `storage_key`, 삭제·손상·비활성·격리 상태, 다운로드 정책 차단 여부를 확인하고, 이러한 상태에 해당하면 다운로드를 차단하도록 정의한다. 따라서 후처리 결과는 단순 부가 기능이 아니라, 실제 파일 접근 가능 여부와 연결되는 정책 값으로 관리되어야 한다.

### 바이러스 스캔 완료 전 다운로드 허용 정책

본 시스템은 **바이러스 스캔이 완료되기 전에도 파일 다운로드를 허용**한다.  
즉, `FileItem`이 최종 생성되고 저장 확정이 완료되었다면, 바이러스 스캔이 아직 `PENDING` 또는 진행 중 상태이더라도 다운로드 세션 발급 대상이 될 수 있다.

이 정책은 기존 설계의 업로드 파이프라인이 `FileItem` 생성 이후 후처리 큐를 등록하고, 바이러스 스캔 역시 후처리 작업 중 하나로 비동기 수행하도록 정의하고 있기 때문에 자연스럽다. 즉, 스캔 완료를 저장 확정의 선행 조건으로 두지 않고, 저장 완료 이후의 후속 처리로 둔다는 의미다.

다만 스캔 완료 전 다운로드 허용은 “무제한 허용”이 아니라, **스캔 결과가 나오기 전까지는 정상 파일로 간주하되, 감염 또는 스캔 실패 결과가 나오면 즉시 접근 정책에 반영한다**는 원칙으로 운영한다.

### 바이러스 스캔 실패 및 감염 탐지 시 상태값

바이러스 스캔 결과 파일 접근을 차단해야 하는 경우, `FileItem` 상태값은 **`quarantined`**로 정의한다.

`quarantined`는 다음 상황에 사용한다.

- 악성 코드 또는 감염 징후가 탐지된 경우
- 스캔 엔진 오류나 후처리 실패로 인해 파일 안전성을 신뢰할 수 없다고 판단한 경우
- 운영 정책상 격리 대상으로 분류된 경우

다운로드 파이프라인은 이미 사전 검증 단계에서 삭제·손상·비활성·격리 상태를 차단 대상으로 보고 있으므로, `quarantined`는 다운로드 차단 상태로 직접 연결된다. 즉, 파일이 `quarantined`이면 다운로드 세션을 발급하지 않으며 외부 응답은 기존 원칙대로 `404 Not Found`로 통일 처리한다.

정리하면 정책은 다음과 같다.

- 스캔 대기 중: 다운로드 허용
- 스캔 성공: 정상 접근 유지
- 스캔 실패 또는 감염 탐지: `quarantined` 전환 및 다운로드 차단

### 썸네일 및 미리보기 생성 실패 시 사용자 경험 정책

이미지 썸네일 생성과 텍스트/PDF 미리보기 캐시는 기존 업로드 후처리 단계에 포함되며, 업로드 직후 클라이언트에도 후처리 상태를 제공하도록 되어 있다. 따라서 썸네일 또는 미리보기 생성 실패는 파일 자체 저장 실패로 취급하지 않고, **후처리 실패로만 취급**한다.

본 시스템은 썸네일 또는 미리보기 생성 실패 시 사용자 화면에서 **플레이스홀더를 표시**한다.  
즉, 다음 정책을 따른다.

- 원본 파일 자체는 정상 저장되었으면 다운로드 가능
- 썸네일 생성 실패 시 기본 아이콘 또는 대표 플레이스홀더 이미지 표시
- 미리보기 생성 실패 시 미리보기 영역에 실패 안내 또는 기본 플레이스홀더 표시
- 사용자는 여전히 파일명, 크기, 업로드 시각 등 메타데이터를 확인하고 원본 다운로드를 수행할 수 있음

이 정책은 미리보기/썸네일을 편의 기능으로 취급하고, 원본 파일 접근성과 분리하기 위한 것이다. 따라서 썸네일·미리보기 실패만으로 파일을 차단 상태로 전환하지 않는다.

### 상태값 및 접근 정책 연결 원칙

후처리 결과와 파일 접근 정책은 다음 원칙으로 연결한다.

- `FileItem` 저장 확정 완료 후 후처리는 비동기로 수행한다.
- 바이러스 스캔 대기 중에는 다운로드를 허용한다.
- 바이러스 스캔 실패 또는 감염 탐지 시 `FileItem`을 `quarantined`로 전환한다.
- `quarantined` 상태 파일은 다운로드 파이프라인의 사전 검증에서 차단한다.
- 썸네일·미리보기 생성 실패는 파일 접근 차단 사유가 아니다.
- 썸네일·미리보기 실패 시 UI에서는 플레이스홀더를 표시한다.

### 클라이언트 상태 표시 권장안

업로드 완료 후 상태 조회 응답 또는 파일 메타데이터 응답에는 후처리 상태를 별도로 포함하는 것이 바람직하다. 기존 문서도 업로드 결과와 후처리 상태를 클라이언트에 제공하도록 정의하고 있다.

권장 예시는 다음과 같다.

{  
  "fileItemId": 987,  
  "scanStatus": "PENDING",  
  "previewStatus": "FAILED",  
  "thumbnailStatus": "FAILED",  
  "accessStatus": "DOWNLOADABLE"  
}

감염 또는 스캔 실패 시 예시는 다음과 같다.

{  
  "fileItemId": 987,  
  "scanStatus": "FAILED",  
  "fileStatus": "QUARANTINED",  
  "accessStatus": "BLOCKED"  
}

위 JSON 필드명은 구현 예시이며, 실제 필드명은 프로젝트의 `FileItem` 메타데이터 구조에 맞게 조정할 수 있다. 다만 정책상 의미는 다음처럼 유지되어야 한다.

- 스캔 결과와 접근 차단 상태는 분리 가능하되 일관되게 연결할 것
- 미리보기 실패는 다운로드 가능 여부와 분리할 것
- 격리 상태는 다운로드 차단과 직접 연결할 것

### 운영 원칙 요약

본 시스템의 후처리 결과와 파일 접근 정책은 다음 원칙을 따른다.

- 바이러스 스캔 완료 전 파일 다운로드는 허용한다.
- 스캔 실패 또는 감염 탐지 시 파일 상태는 `quarantined`로 전환한다.
- `quarantined` 파일은 다운로드 세션 발급 전 사전 검증 단계에서 차단한다.
- 썸네일·미리보기 생성 실패는 파일 저장 실패나 접근 차단으로 보지 않는다.
- 썸네일·미리보기 생성 실패 시 사용자 화면에는 플레이스홀더를 표시한다.
- 후처리 상태는 클라이언트가 조회 가능하도록 응답에 포함한다.
