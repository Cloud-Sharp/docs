    정리 배치·만료 처리 운영 규칙

    1. 목적¶

    단순 삭제가 아닌 네 가지 핵심 목표를 달성하기 위한 운영 규칙이다.

    목표 설명
    세션 상태 정합성 유지 비정상 종료된 세션이 중간 상태로 잔존하지 않도록 보정
    예약 자원 회수 실패·취소·만료 세션의 FileReservation 및 quota 반영
    임시 저장소 누수 방지 orphan file이 임시 디렉터리에 축적되지 않도록 정리
    운영 장애 조기 감지 반복 실패 시 알림을 통해 운영자 개입 유도

    2. EXPIRED 전환 배치¶

    실행 주기¶

    • 기본 5분 주기, 운영 환경에 따라 조정 가능

    대상¶

    - 만료 시각이 지난 UploadSession
- 만료 시각이 지난 FileReservation
- 업로드 중단 후 장시간 미진행 세션
- 정상 종료 없이 방치된 세션

    동작¶

    • 만료 조건 충족 시 → EXPIRED 상태 전환
    • 관련 예약 자원 → 함께 만료/해제 처리

    3. Orphan File 정리¶

    스캔 범위¶

    구분 포함 여부
    업로드 중 생성된 청크 파일 ✅
    병합 완료 전 임시 파일 ✅
    finalize 이전 임시 저장 파일 ✅
    최종 저장소 영역 ❌ 대상 아님

    정리 범위는 임시 디렉터리에 한정한다.

    판정 기준 (하나라도 해당 시 orphan)¶

    1. 참조하는 UploadSession이 존재하지 않는 파일
2. 연결된 세션이 FAILED / ABORTED / EXPIRED 종료 상태인 파일
3. 생성·수정 시각 기준 24시간 초과 임시 파일
4. 세션 실패 + 예약 해제 완료 → 복구 대상 아닌 파일

    핵심: "세션이 없는 파일"뿐 아니라 "세션은 있었지만 이미 종료되어 유지 이유가 없는 파일"까지 포함

    4. 정리 실패 시 재시도 정책¶

    실패 발생
  ├─ 즉시 반복 재시도 ❌ (하지 않음)
  ├─ 실패 사실 로그 기록 ✅
  └─ 다음 정기 배치(5분 후)에서 동일 대상 재점검 ✅
    • 별도 지수 백오프 없음 — 배치 주기 자체가 재시도 간격
    • 파일시스템 잠금, 외부 저장소 일시 장애, 메타데이터 반영 지연 등 일시적 오류에 대한 과도한 재시도 방지

    5. 장애 알림 기준¶

    트리거: 동일 대상/배치에서 2회 연속 실패¶

    횟수 대응
    1회 실패 로그만 기록 (일시적 오류 가능성 고려)
    2회 연속 실패 운영 알림 발송

    알림 대상 예시¶

    - 동일 orphan file 삭제 2회 연속 실패
- 동일 세션 EXPIRED 전환/종료 보정 2회 연속 실패
- 임시 디렉터리 정리 배치 전체 2회 연속 비정상 종료

    6. FileReservation 해제 규칙¶

    해제 트리거 (세션 비정상 종료 시)¶

    UploadSession → FAILED   ─┐
UploadSession → ABORTED  ─┼─→ 예약 즉시 해제 + quota 차감 반영
UploadSession → EXPIRED  ─┘

    처리 내용: - 예약 상태 → 종료 전환 - storage_reserved_bytes → 차감 - storage_used_bytes → 변경 없음 (실제 파일 확정이 없으므로)

    정상 완료 시 (finalize)¶

    finalize 트랜잭션 내에서:
  예약 상태 → CONSUMED
  reserved  → 감소
  used      → 증가
    시나리오 reserved used 예약 상태
    실패 / 취소 / 만료 감소 변동 없음 해제(종료)
    정상 완료 (finalize) 감소 증가 CONSUMED

    7. 전체 운영 원칙 한눈에 보기¶

    ┌─────────────────────────────────────────────────────────┐
│  1. 만료 전환 배치        → 5분 주기 실행                   │
│  2. orphan file 스캔      → 임시 디렉터리 한정              │
│  3. orphan file 판정      → 미연결 + 종료 세션 + 24h 초과   │
│  4. 정리 실패 재시도      → 즉시 재시도 없음, 다음 배치       │
│  5. 장애 알림             → 2회 연속 실패 시 발송           │
│  6. 예약 해제             → 세션 종료 시 즉시 + quota 반영  │
└─────────────────────────────────────────────────────────┘
